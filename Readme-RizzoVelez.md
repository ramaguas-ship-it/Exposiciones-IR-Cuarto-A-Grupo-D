# Informe de Exposiciones

## EQUIPO C: ZenTao

### Expositor Castro

**Teoría:**

* ¿Qué es ZenTao?: Plataforma de gestión de procesos de software y requisitos de proyectos.
* ZenTao está adaptado a Scrum.
* Herramientas: Backlog, Kanban, Sprints y Wiki.
* Procesos: Elicitación, análisis, especificación, validación, gestión de cambios y trazabilidad.

**Práctica:**

* Creación de un programa.
* Creación de historia de usuario.

### Expositor Vera

**Teoría:**

* Aplicabilidad en el proyecto SIMPA.
* Uso de módulos de IA en el proyecto.
* Uso del backlog en la fase de análisis.
* Uso de Wiki para documentar procesos y cambios.
* ZenTao complementa al ingeniero de requisitos.

**Práctica:**

* Registro de productos.

### Expositor Mora

**Teoría:**

* Comparación entre ZenTao, Jira, Trello y Azure DevOps.
* Ventajas: Integración con Git, gratuito, curva de aprendizaje baja y Wiki integrada.
* Limitaciones: Dependencia de Internet.

**Práctica:**

* Creación de proyecto.
* Enlazar proyecto.

### Expositor Villafuerte

**Teoría:**

* Aplicación de ZenTao en Ingeniería de Requerimientos.
* Documentación de requisitos de inicio a fin.
* Complemento para la gestión de requisitos.

**Práctica:**

* Guardar acciones de usuario.
* Creación de Sprint.
